## 一、品牌定位与视觉

### 1.1 品牌主张

| 中文 | 驰于阡陌，自在奔跑 |
|------|-------------------|
| 英文 | Stride in Moor, Run at Ease |
| 价值宣言 | **驰陌，用陪伴让跑步更健康** |

**核心价值：**

- **不卷速度** → 专注健康跑（配速/心率/步频合理区间）
- **有跑友陪伴** → 减少孤独感，提高运动意愿
- **数据对比** → 客观进步，而非盲目追求成绩

### 1.2 产品定义

| 维度 | 说明 |
|------|------|
| 目标用户 | 跑步爱好者（初级~中级），有运动社交需求 |
| 核心功能 | 路线分享 → 导入收藏 → 影子陪跑 → 数据对比 |
| 差异化 | 不做赛事/跑团，围绕"路线"建立社交闭环 |
| 健康跑标准 | 配速合理区间 / 心率有氧区间 / 步频 170+ |

### 1.3 品牌视觉规范

| 元素 | 建议 | 理由 |
|------|------|------|
| 主色调 | 🟢 绿色（自然）+ 🟠 橙色（活力） | 代表健康、运动、活力 |
| 字体 | 圆润无衬线（易读友好） | 传递亲和力、非竞技感 |
| 图标 | 脚印 + 路线 + 对话气泡 | 象征跑步、路线、陪伴 |
| 海报风格 | 人像剪影 + 路线地图 + 数据对比卡片 | 社交货币，适合朋友圈传播 |

## 二、用户痛点与解决方案

```
用户痛点                     → 驰陌解决方案
├── "我想跑但怕受伤"        → 数据诊断 + 配速建议
├── "一个人跑没动力"        → 影子陪跑 + 实时语音鼓励
├── "不知道练得是否有效"    → AI 诊断报告
├── "跑友难找"              → 同城路线挑战
└── "跑步数据没用"          → 可视化对比 + 社交证明
```

**核心信息：**
> 在驰陌，我们不追求 PB，只追求健康、可持续的跑步。

## 三、健康跑标准体系

| 维度 | 健康跑标准 |
|------|-----------|
| 配速 | 低于个人心率/步频阈值 10% |
| 心率 | 维持在最大心率的 60-75%（有氧区间） |
| 步频 | 170+（高效跑步） |
| 步幅 | 适中（避免过大导致受伤） |
| 跑后恢复 | 次日无明显疲劳感 |

## 四、社交圈构建策略

### 4.1 跑友关系分级

```
跑友等级（由浅入深）：
┌─────────────────────────────────────┐
│ 1️⃣ 陌生人    → 挑战邀请（匿名）        │
│ 2️⃣ 挑战者    → 公开排名（有成绩展示）    │
│ 3️⃣ 多次陪跑  → 好友关系（可见动态）      │
│ 4️⃣ 固定跑友  → 定期约跑（建立习惯）      │
│ 5️⃣ 跑圈成员  → 本地社群（同城跑迹）      │
└─────────────────────────────────────┘
```

### 4.2 跑友社交工具

| 工具 | 功能 | 社交价值 |
|------|------|---------|
| 陪跑报告 | 跑步后自动生成对比 + 建议 | 分享朋友圈的素材 |
| 挑战证书 | 挑战成功生成徽章/勋章 | 身份标识，增强成就感 |
| 跑友动态 | 跑友跑步记录分享 | 发现新跑友、激发运动欲 |
| 跑圈地图 | 同城热门跑迹 + 活跃跑友 | 线下约跑、建立本地社群 |
| 健康跑小组 | 基于心率/配速区间的群组 | 健康跑交流、互相监督 |

## 七、API 开放战略

### 7.1 API 开放目标

| 角色 | 价值 |
|------|------|
| 硬件厂商 | 用"驰陌数据"提升设备价值（卖设备时卖"社交/数据"） |
| 用户 | 一键接入数万台设备，无需二次开发 |
| 平台 | 快速获得数万台设备数据接入，提升"影子陪跑"体验 |

### 7.2 API 设计原则

| 原则 | 说明 |
|------|------|
| 标准化 | 统一设备接入协议、数据格式（心跳/步频/轨迹） |
| 轻量化 | SDK 体积 <500KB，支持一键安装 |
| 稳定性 | 99.9% 可用性，断线自动重连 |
| 可扩展 | 预留未来设备类型（智能鞋垫、指环）扩展接口 |

### 7.3 目标厂商类型

| 厂商类型 | 代表品牌 | 接入价值 |
|---------|---------|---------|
| 运动指环 | 小米指环、华为指环、Amazfit 指环 | 极致轻量化，用户基数大 |
| 运动手环 | 华为手环、Apple Watch、小米手环 | 主流设备，生态完善 |
| 智能手表 | Garmin、Polar、佳明 | 专业跑者，数据丰富 |
| 第三方厂商 | 华强北 ODM 厂商 | 价格敏感，需要第三方平台提升价值 |

### 7.4 商业模型

| 模式 | 说明 |
|------|------|
| 免费 SDK | 厂商免费使用，提供"驰陌数据"提升设备价值 |
| 设备分成 | 平台推荐"驰陌认证跑鞋/手表"，销售分成 |
| 数据聚合 | 数万台设备数据 = 高质量"健康跑数据集"，未来可商业化 |
| B 端合作 | 平台对接跑者保险、装备电商等 B 端服务 |
