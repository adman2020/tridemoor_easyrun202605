# 跑境体系 · 修仙境界 × 运动能力

> *《凡人修仙传》有云：「道不可轻传，留一缕仙缘待有缘人。」*
> **每一条跑步和骑行数据都留在路上，后来者可以借你的「道标」突破自我。**

为 StrideMoor（踏歌）社交跑步+骑行 App 的角色能力分级标准。
App「我的」页面 → 境界入口，展示**跑步十三境**与**骑行十三境**两套独立勋章墙（左右滑动切换），当前进度按各自运动独立计算。
境界映射实际运动能力（跑步：完赛距离/全马成绩；骑行：单次距离/百公里均速）。
依据《凡人修仙传》原著 13 大境界正版排序，两套勋章共用同一命名体系。

**勋章资源位置（跑步）：** `assets/badges/paojing/`
**勋章资源位置（骑行）：** `assets/badges/qixing/`
**规格：** 各 26 枚（13 点亮 + 13 虚化），每枚 4 种尺寸（80/120/160/200px）
**墙预览：** `wall_lit.png` / `wall_dim.png`（4+4+5 三行布局）

## 四、数据规则

### 4.1 炼气 ~ 化神 数据规则

- 只记录 **距离** 和 **时间**，不关注配速高低
- 数据库 `avg_pace` 字段可为 NULL
- 前端显示：显示距离和时间，不显示配速（或显示为 "— /km"）
- 测试数据生成脚本：按境界生成对应距离的跑步记录（5km/10km/半马/全马），时间在合理范围内随机即可

### 4.2 练虚及以上 数据规则

- 必须记录 **全马成绩**（时间 + 距离 42.195km + 配速）
- `avg_pace` 字段存储为 **秒/公里（INT）**
- 配速严格等于 `total_time * 1000 / (60 * total_distance)` 的整数结果
- 全马成绩不得违反对应境界的闭区间上限

### 4.3 跑步记录类型

| 记录类型 | 距离范围 | 适用境界 |
|---------|---------|---------|
| 入门跑 | 2.5 ~ 5 km | 炼气~筑基 |
| 日常跑 | 5 ~ 15 km | 筑基~结丹 |
| 10km跑 | 10 km（精确） | 结丹考核 |
| 半马跑 | 21.0975 km（精确） | 元婴考核 |
| 全马跑 | 42.195 km（精确） | 化神考核，练虚及以上正赛 |

### 4.4 生成测试数据时的约束

1. **境界越高 ≠ 跑得越快**——境界决定完赛能力，练虚及以上才谈配速
2. **化神及以下**：首要任务是跑完全程，配速不设限
3. **练虚及以上**：配速必须符合境界区间上限
4. **同一境界可以速度不同**：如两个化神角色，一个4:30/km一个5:30/km都合理
5. **分组合理**：每次生成测试数据时，按境界分布产生对应距离和配速的数据

### 5.5 自动判境与晋升机制

#### 晋升条件（多维度判定）

晋升不再只看距离/配速，还需完成社交互动（伴跑）和挑战达标，构建完整的跑者成长体系。
每次达成任一条件（跑完、伴跑完成、挑战成功、发动态）自动触发判境检查。

**判境五维度：**

| 维度 | 含义 | 适用境界段 |
|------|------|----------|
|  **跑步距离** | 最佳单次跑步距离 | 下五境（炼气→化神） |
| ⏱ **全马配速** | 全马成绩换算（秒/km） | 上八境（练虚→道祖） |
|  **在线伴跑** | AI 伴跑完成次数 | 炼气→筑基 |
|  **挑战成功** | 挑战排行榜用户成功次数 | 筑基→元婴 |
|   **发布动态** | 发布到跑友动态的次数 | 元婴→道祖 |

**晋升条件表：**

| 当前境界 | 晋升下一境条件 | 触发时机 |
|---------|-------------|---------|
| 炼气→筑基 | 5km + **1次在线伴跑** | 跑后伴跑后检查 |
| 筑基→结丹 | 10km + **2次挑战成功** | 跑后挑战后检查 |
| 结丹→元婴 | 半马 + **5次挑战成功** | 跑后挑战后检查 |
| 元婴→化神 | 全马 + **发1次动态** | 跑后发动态后检查 |
| 化神→练虚 | 全马 + 发1次动态 + **配速≤4:15/km** | 跑后发动态后检查 |
| 练虚→合体 | 全马 + 发1次动态 + **配速≤3:55/km** | 跑后发动态后检查 |
| 合体→大乘 | 全马 + 发1次动态 + **配速≤3:33/km** | 跑后发动态后检查 |
| 大乘→真仙 | 全马 + 发1次动态 + **配速≤3:19/km** | 跑后发动态后检查 |
| 真仙→金仙 | 全马 + 发1次动态 + **配速≤3:12/km** | 跑后发动态后检查 |

> 注：元婴→化神是分水岭——从此开始**配速**成为关键判境指标。

#### 进度计算算法（木桶效应）

多维度的进度取**最小值**，短板决定最终进度：

`
index 0 (炼气→筑基): min(距离/5km,  伴跑次数/1)
index 1 (筑基→结丹): min((距离-5km)/5km,  挑战次数/2)
index 2 (结丹→元婴): min((距离-10km)/11.1km, 挑战次数/5)
index 3 (元婴→化神): min((距离-21.1km)/21.1km,  动态数/1)
index 4+ (化神→道祖): 配速区间占比（仅配速维度）
`

示例：跑了 8km、0 次伴跑 → min(8/5=1.0, 0/1=0.0) = 0%，因为伴跑没完成。

#### 触发时机详细

| 事件 | 动作 | 说明 |
|------|------|------|
| 跑步完成 | 调用 POST /user/paojing/check | 携带本次距离，更新最佳单次距离 |
| 伴跑完成 | 调用同接口 | 携带伴跑完成标记，计数+1 |
| 挑战成功 | 调用同接口 | 携带挑战成功标记，计数+1 |
| 发动态 | 调用同接口 | 检查动态数是否≥1 |
| 自动 | GET /user/paojing | 前端 API 每次都返回最新判境结果 |

> 注：伴跑、挑战、及其他待定条件的数值均待最终确认。

#### 维度说明

| 维度 | 含义 | 适用境界段 |
|------|------|----------|
| 🏃 **跑步距离** | 单次/累计跑步距离 | 下五境（炼气→化神） |
| ⏱ **全马配速** | 全程马拉松完赛时间 | 上八境（练虚→道祖） |
| 👥 **在线伴跑** | AI 伴跑（选择同线路跑过的真人数据），无需对方同意 | 低阶引导社交 |
| 🏆 **挑战成功** | 挑战线路排行榜上的任何人（无需对方同意），可选择任一指标：速度/平均心率/平均步频/平均步幅，达到或超越即成功 | 中高阶证明实力 |
| ❓ **待定** | 后续补充（连续跑、勋章收集等） | 高阶 |

#### 伴跑与挑战的区别

| 维度 | 🏃 伴跑 | 🏆 挑战 |
|------|---------|---------|
| 目标 | 陪伴完成跑步 | 击败某人的数据 |
| 对手 | 同线路跑过的任何人（AI 化身） | 线路排行榜上的任何人 |
| 是否需同意 | ❌ 不需要 | ❌ 不需要 |
| 跑起来效果 | 对方的数据在你跑步过程中同步显示 | 对方跑过的数据在你跑时实时对比 |
| 判定指标 | 无特定指标，完成即算 | 可选：速度 / 平均心率 / 平均步频 / 平均步幅 |
| 成功条件 | 跑完全程 | 所选指标达到或超越对手 |
| 是否计入晋升 | ✅ 炼气→筑基条件（需1次） | ✅ 筑基→结丹（需2次）、结丹→元婴（需5次） |

**设计理念：** 不需要对方在线，不需要对方同意。你的每一条跑步数据都留在路上，成为后来者的「仙缘」。修仙路上不孤单，前人留下的足迹就是你的指引。

#### 晋升效果

1. 更新用户境界字段
2. 颁发对应勋章（首次获得）
3. 系统自动发一条动态：「恭喜 XXX 突破至 XXX 境界！」
4. 推送通知到好友动态

#### 头像徽章显示规则

- 头像右下角只显示**已达成境界**的点亮小徽章
- 虚化境界**不显示**在头像旁
- 当前境界 + 已过境界 = 全部点亮显示在勋章墙
- 示例（结丹期用户）：
  - 头像右下角：✨ **丹**（当前境，点亮）
  - 勋章墙：✨气 ✨筑 ✨丹 🌫婴 🌫化 ...（前三亮，后全虚）

#### 数据存储建议

```dart
// User 模型新增字段

realmBadges: List<String>?  // 已获得勋章列表: ['气','筑','丹',...]
realmProgress: double?      // 晋升进度 0.0~1.0

// 条件追踪
```

## 七、骑行十三境（双轨独立·同一命名体系）

### 7.1 双轨设计总览

> 跑步和骑行共用「炼气→道祖」十三境命名体系，但两套勋章**独立晋升、独立点亮**。
> 用户主页显示的主境界 = 跑步/骑行中的**更高者**。

| 维度 | 跑步十三境 | 骑行十三境 |
|------|-----------|-----------|
| 命名 | 炼气→道祖 | 炼气→道祖（完全一致） |
| 晋升 | 跑步数据独立计算 | 骑行数据独立计算 |
| 勋章墙 | 第1页 🏃 跑步十三境 | 第2页 🚴 骑行十三境 |
| 头像徽章字 | 主境界字（取更高者） | 主境界字（取更高者） |
| 头像圈色 | 🔵 仅跑步 / 🟠 仅骑行 / 🟢 双修 | 同上 |

**双修额外成就（第3页勋章墙）：**
- 🔄 初入双修：跑步+骑行均筑基
- ⚔️ 双剑合璧：跑步+骑行均元婴
- 🐉 龙骑虎步：跑步+骑行均化神
- ☯️ 道法自然：跑步+骑行均练虚
- ⭐ 破界飞升：跑步+骑行均真仙
- 🌈 无极之境：跑步+骑行均道祖

### 7.2 骑行十三境判境标准

#### 下五境（距离）

| 境界 | 🏃 跑步判境 | 🚴 骑行判境 | 说明 |
|------|-----------|-----------|------|
| **炼气** | 首次跑步 | 首次骑行 | 任何骑行记录即入 |
| **筑基** | 单次跑完 5km | 单次骑完 20km | 4x跑步距离 ≈ 相近消耗 |
| **结丹** | 单次跑完 10km | 单次骑完 40km | 城市通勤一趟 |
| **元婴** | 单次跑完 半马(21.1km) | 单次骑完 80km | 周末休闲骑距离 |
| **化神** | 单次跑完 全马(42.2km) | 单次骑完 160km | 百英里骑行 |

**判距说明：**
- 骑行距离 (km) 判断取**单次最佳成绩**（与跑步逻辑一致）
- 手机 GPS 记录即可判境，不依赖硬件
- 数据存储字段：`cyclingBestDistance`

#### 上八境（均速）

| 境界 | 🏃 跑步（全马配速） | 🚴 骑行（100km均速） | 骑行条件说明 |
|------|------------------|---------------------|-------------|
| **练虚** | 全马 ≤ 3:00:00（4:15/km） | 100km 均速 ≥ **33.3 km/h** | 100km 3h内完赛 |
| **合体** | 全马 ≤ 2:45:00（3:55/km） | 100km 均速 ≥ **36.4 km/h** | 100km 2h45m内 |
| **大乘** | 全马 ≤ 2:30:00（3:33/km） | 100km 均速 ≥ **40.0 km/h** | 100km 2h30m内 |
| **真仙** | 全马 ≤ 2:20:00（3:19/km） | 100km 均速 ≥ **42.9 km/h** | 100km 2h20m内 |
| **金仙** | 全马 ≤ 2:15:00（3:12/km） | 100km 均速 ≥ **44.4 km/h** | 100km 2h15m内 |
| **太乙** | 全马 ≤ 2:10:00（3:05/km） | 100km 均速 ≥ **46.2 km/h** | 100km 2h10m内 |
| **大罗** | 全马 ≤ 2:05:00（2:58/km） | 100km 均速 ≥ **48.0 km/h** | 100km 2h05m内 |
| **道祖** | 全马 < 2:00:00（2:50/km） | 100km 均速 > **50.0 km/h** | 100km 2h内 |

**判速说明：**
- 100km 为基准计时距离（不要求单次恰好 100km，取距离 ≥ 100km 的记录计算均速）
- 百公里均速 = 总距离(km) / 总用时(h)，取最佳成绩
- 与跑步上八境逻辑一致：从「完成距离」转向「速度/功率」
- 数据存储字段：`cyclingBestSpeed`、`cyclingBestTimed_100km`

### 7.3 骑行判境数据规则

#### 炼气 → 化神 数据规则

与跑步版一致：
- 只记录距离和时间，不关注均速高低
- 数据库 `avg_speed` 字段可为 NULL（跑步版 `avg_pace` 同理）
- 前端显示：显示距离和时间，不显示均速（或显示 


#### 练虚及以上 数据规则

- 必须记录 100km 成绩（距离 >= 100km 的骑行记录）
- 均速严格计算：avg_speed = total_distance_km / total_time_h
- 百公里均速不得低于对应境界下限

#### 骑行记录类型

| 记录类型 | 距离范围 | 适用境界 |
|---------|---------|---------|
| 通勤骑 | 5 ~ 30 km | 炼气~筑基 |
| 休闲骑 | 30 ~ 80 km | 筑基~结丹 |
| 长途骑 | 80 ~ 160 km | 元婴~化神 |
| 百公里计时 | >= 100 km | 练虚及以上正赛 |
| 百英里骑 | >= 160 km | 化神考核 |
