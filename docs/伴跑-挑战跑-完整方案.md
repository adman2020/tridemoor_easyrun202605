## 一、核心概念

### 伴跑（Companion Run）
选一条已收藏的跑友跑迹，跟着对方的幽灵数据跑。陪跑性质，跑后只展示对比数据，不判胜负。

### 挑战跑（Challenge Run）
选一条已收藏的跑友跑迹，发起挑战。跑完后系统自动对比双方数据，判胜负，更新记录。

**关键规则：**
- 不需要对方同意 —— 全程单人异步，对方的跑迹作为幽灵素材
- 所有伴跑/挑战跑的跑迹来源：**已收藏的跑友跑迹**
- 跑友跑迹来自：跑迹广场、跑友动态等处收藏

## 二、用户流程

### 入口：跑步准备页

```
跑步准备页
├── [独自跑]（默认）
├── [伴跑] → 选 GhostMode → [选择跑迹] → 已收藏的跑友跑迹列表 → 开跑
└── [挑战跑] → 选挑战指标 → [选择跑迹] → 已收藏的跑友跑迹列表 → 开跑
```

### 跑步准备页 UI 结构（示意）

```
┌──────────────────────────┐
│  跑步准备                 │
│                          │
│  方式：○ 独自跑           │
│        ○ 伴跑             │
│        ○ 挑战跑           │
│                          │
│  [选择跑迹]  ← 选中伴跑/挑战跑时出现 │
│                          │
│  [开始跑步]               │
└──────────────────────────┘
```

### 选择跑迹

点击「选择跑迹」后弹出收藏的跑友跑迹列表，每条包含：
- 跑友昵称 + 头像
- 路线名称
- 原始跑步数据（距离、配速、用时、步频、步幅、心率）
- 选中后：路线 + 对手数据 + GPS 轨迹点全部就绪

---

## 三、伴跑模式（GhostMode）

| 模式 | 说明 |
|:----|:------|
| **真实复刻** | 严格还原对手真实的配速变化，幽灵按原数据跑 |
| **匀速目标** | 以对手的平均配速匀速前行 |
| **兔式超前** | 比对手快 5% 开始跑，前快后慢 |
| **龟兔模式** | 前快后慢，后半段用力冲 |
| **目标挑战** | 选择维度（配速/心率/步频）挑战自己的极限 |

---

## 四、挑战指标 & GhostMode 规则

### GhostMode 默认值
- 伴跑：用户可选择任意 GhostMode
- 挑战跑：GhostMode **固定为「真实复刻」**

### 挑战指标（ChallengeMetric）

| 指标 | 单位 | 说明 |
|:----|:----|:------|
| 配速 | min/km | 同路线比配速 |
| 心率 | bpm | 同配速下心率更低者优 |
| 步频 | spm | 同配速下步频更优者胜 |
| 步幅 | m | 同配速下步幅更大者胜 |

## 五、跑中体验

> 伴跑与挑战跑跑中基本一致，仅跑后行为不同。


### 语音播报（按「我的-播报设置」执行）

**频次设置：**
- 距离触发：每 200m / 500m / 1km
- 时间触发：每 5min / 10min
- 异常触发：仅心率超标/配速偏离时

**播报项（13 项）：**
基础：配速、距离、用时、心率、步频、步幅
对手相关：差距（lag）、对手配速（opponent_pace）、配速偏差（pace_deviation）
状态：目标状态（goal_status）、爬升提醒、打鸡血、冲刺提醒

**语音风格：** 标准 / 江湖 / 教练 / 毒舌

### 对手幽灵数据展示

跑步页面需要显示对手实时信息：
- 对手的总距离和我的差距
- 对手当前配速 vs 我的配速
- 分段对比颜色指示（我在前/落后/平手）

## 六、跑后行为

### 1. 双人对比报告（完成页）

对比维度（至少包含）：

| 维度 | 对比方式 |
|:----|:---------|
| 总用时 | 谁快谁胜 |
| 平均配速 | 数值对比 |
| 配速走势 | 分段折线对比 |
| 心率对比 | 平均值 + 分段 |
| 步频/步幅 | 数值对比 |
| 分段胜负 | 每公里标记谁领先 |

### 2. 伴跑 — 无记录

**伴跑不保存跑步记录**，相当于学习训练。
- 不产生新的 Run 记录
- 不更新挑战记录
- 仅更新热度

### 3. 挑战跑 — 更新挑战记录

- 路径：`我的 → 挑战记录` (`/profile/challenges`)
- 已存在 `ChallengeHistoryPage` 页面
- 挑战跑完成后写入新的挑战记录

### 4. 更新轨迹热度（纯统计功能）

**设计定位：** 纯统计展示，不涉及复杂互动逻辑。

**统计维度（展示字段，目前用一个总值累计，后续可拆分）：**
- 被收藏次数（总计）
- 伴跑次数（被作为别人伴跑）
- 被关注次数
- 被挑战次数（成功/失败）

**实现方式：**
- 后端已有 `Route.Popularity`（路线热度）和 `Run.HeatCount`（跑步记录热度）
- 被伴跑一次 → `HeatCount +1`
- 被挑战一次 → `HeatCount +1`
- 将来要细分 4 个维度再加独立计数
- 热度高的跑迹在排序上有提升
