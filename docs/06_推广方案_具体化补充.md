## 一、内容营销：可复用的内容SOP

### 1.1 内容支柱体系

将内容分为 5 根支柱，每根支柱对应固定的内容格式和发布节奏：

| 支柱 | 定位 | 内容形式 | 平台 | 频率 | 衡量指标 |
|------|------|---------|------|------|---------|
| 🏃 **跑者故事** | 情感共鸣+品牌温度 | 人物专访图文/短视频 | 小红书/公众号/B站 | 每周1篇 | 点赞率>5%、收藏率>3% |
| 📊 **数据科普** | 专业感+教育用户 | 数据对比图文/图解 | 小红书/知乎 | 每周2篇 | 完读率>60%、评论>20 |
| 🗺️ **跑迹推荐** | 实用价值+地域渗透 | 路线攻略图文/GIF动图 | 小红书/抖音 | 每周1篇 | 收藏率>8%、分享率>5% |
| 🏆 **挑战热点** | 社交驱动+UGC激励 | 排行榜海报/榜单 | 社群/朋友圈 | 每两周1期 | 参与率>10%、截图传播>50次 |
| 🎮 **境界体系** | 游戏化+话题性 | 境界对应表/成绩对照/修仙梗图 | 小红书/社群/朋友圈 | 每月1波 | 自传播率、话题讨论量 |

### 1.2 可复用的内容模板

#### 模板A：跑者故事（千字图文版）

```
标题公式：{数字} + {人群标签} + {变化量} + {情绪钩子}
示例：    "34岁程序员的跑步逆袭：从跑800米喘到完成半马，我做对了3件事"

结构：
【开头】- 1-2段制造悬念（以前的困境）
  "去年体检报告出来，脂肪肝+高血脂+颈椎病，我站在体重秤上看了看..."

【正文】- 时间线叙事+关键节点
  "第1个月：跑800米就喘（配附真实App截图）"
  "第3个月：第一次完成5公里（重点：用了伴跑模式）"
  "第6个月：完成半马（配App数据对比图）"

【干货】- 3条可复制经验（自然植入产品功能）
  "1. 找条路线跟着跑，比瞎跑强10倍 → 伴跑模式"
  "2. 看数据对比才知道自己进步 → 对比报告"
  "3. 有人陪跑（哪怕是影子），坚持率翻倍"

【结尾】- 互动式CTA
  "你最快的一次5公里配速是多少？评论区晒数据"
  或 "想挑战这条路线？搜'深圳人才公园5K'加入"
```

#### 模板B：数据科普卡片（小红书信息图）

```
标题公式：{反常识} + {数据} + {结论}
示例：    "跑了3个月配速没变？问题不在腿，在步频"

卡片1（封面）：大字标题 + 品牌logo + 绿色/橙色主色调
卡片2：痛点场景图 + 数据对比（慢vs快）
卡片3：解决方案步骤 + App截图配文
卡片4：数据对比结果（改造前vs改造后）
卡片5：行动号召 + 话题标签
  #健康跑 #跑步干货 #跑步新手 #运动数据

固定尾标：
  底部小字 "数据来源：驰陌/StrideMoor·你的影子跑友"
```

#### 模板C：跑迹推荐（本地化攻略）

```
标题公式：{城市} 最{形容词}的{数字}条跑步路线，{亮点}
示例：    "深圳最适合夜跑的5条路线，人少景美还有路灯"

内容结构：
1. 路线概览（距离/爬升/难度星级）
2. 跑友实测评价（引用2-3条App内评测）
3. 最佳跑步时段建议
4. 周边设施（存包/饮水/停车）
5. 地图卡片 + 轨迹示意图
6. 互动问题 "你在这条路线的最佳配速是多少？"
```

## 三、裂变增长机制详细设计

### 3.1 核心裂变回路

```
用户A跑完 → 生成对比/挑战卡片
                ↓
          分享到微信/朋友圈/群
                ↓
（B看到卡片）→ 长按扫码 → 打开App
                ↓
          注册 → 自动收藏该路线
                ↓
          B用伴跑模式跑一次 → 完成
                ↓
          自动生成"追上了A"对比报告
                ↓
          B分享到朋友圈 → A收到通知
                ↓
        循环继续 → 更多C/D加入
```

### 3.2 三种裂变机制

#### 机制一：路线挑战卡（核心裂变）

**触发点：** 用户完成跑步后，生成路线挑战卡
**卡片内容：**
- 路线地图缩略图
- 用户成绩（配速/用时/步频）
- 大按钮：「挑战TA」
- 二维码（深度链接到路线详情页）

**裂变动力：** 看到朋友的成绩，想"我也来试试这条路线"
**转化率预估：** 看卡片→打开App 10-15%，打开→注册 60-70%

#### 机制二：境界突破卡（情感裂变）

**触发点：** 用户晋升新境界时自动生成
**卡片内容：**
- 境界名称+徽章（大字号）
- 用户突破故事：「我从筑基期突破到结丹期！」
- 趣味数据：「累计跑了120公里，完成了15次挑战」
- 小字：「来驰陌测测你的跑者境界」

**裂变动力：** 修仙话题自带传播力，朋友好奇"我是什么境界"
**转化率预估：** 看到→打开App 15-20%

#### 机制三：邀请好友得境界加速（利益裂变）

**机制设计：**

| 邀请人数 | 邀请者奖励 | 被邀请者奖励 |
|---------|-----------|------------|
| 1人 | 专属头像挂件「引路人」 | 新手保护期3天（数据不记入排名，降低门槛） |
| 3人 | 境界加速卡×1（50%晋升经验加成1次） | 首条收藏路线免费解锁 |
| 5人 | 境界加速卡×2 + 专属称号「传道者」 | 免费解锁挑战模式7天 |
| 10人 | 当月VIP会员 | 免费解锁挑战模式7天 |
| 20人 | 实体奖励（跑者周边/速干衣等） | — |

**积分兑换参考（替代方案）：**
- 邀请1人 = 100积分
- 积分可兑换：50积分换1天VIP、200积分换境界加速卡、500积分换实体优惠券

### 3.3 深度链接与转化路径

```
用户在地铁刷朋友圈看到路线挑战卡
  ↓
点击/扫码 → 打开深度链接 `stridemoor://route/xxx`
  ├─ 已安装App → 直接跳转路线详情页 → 注册/登录 → 收藏
  └─ 未安装 → 跳应用商店下载页
         ↓
     安装完成 → 首次打开App → 弹窗"你有一条路线待查看"
         ↓
     授权地理位置 → 看到路线详情 → 系统自动收藏
         ↓
     准备页 → 点击"开始伴跑" → GPS搜星 → 开跑
         ↓
     跑完→生成对比报告→分享→进入下一个裂变循环
```

**转化漏斗参考值：**
```
曝光（朋友圈看到卡片）        100%
   ↓
点击/扫码                   15-25%（取决于卡片设计+文案）
   ↓
到达路线详情页               60-70%（已安装）／30-50%（未安装→下载→首次打开）
   ↓
收藏路线                    40-50%
   ↓
实际开跑（7天内）             20-30%
   ↓
完成跑步+生成报告             15-20%
   ↓
二次分享                     8-12%
```
