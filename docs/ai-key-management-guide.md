# AI 密钥管理 — 功能操作指南

## 概述

AI 密钥管理模块用于统一管理驰陌 App 中各 AI 功能的 **API 密钥、模型调用、日限额**。管理员可在后台一键配置各功能的启用状态、绑定模型和密钥，所有配置即时生效。

## 进入页面

1. 浏览器打开 **https://admin.8keya.com**
2. 按 **Ctrl + Shift + R** 强制刷新（清除旧缓存）
3. 登录管理后台
4. 左侧栏 → **驰陌管理 → AI密钥管理**

## 页面布局

| 表头 | 说明 |
|------|------|
| 功能 | AI 功能名称 |
| 启用状态 | 开关控件，控制功能是否可用 |
| 使用模型 | 该功能调用的模型（如 deepseek-chat） |
| 调用密钥 | 脱敏显示（前4+后4，中间****） |
| 日限额 | 进度条 + 今日已用量/总量 |
| 操作 | 编辑 / 轮换 / 删除 |

## 新增配置

点击 **「新增功能」**，弹出表单：

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| 功能名称 | ✅ | 功能的中文名称 | AI跑步教练 |
| 供应商 | ✅ | 密钥供应商 | DeepSeek / Moonshot / 通义千问 / OpenAI |
| 使用模型 | ✅ | 模型标识 | deepseek-chat / moonshot-v1 / qwen-turbo |
| 调用密钥 | ✅ | 供应商提供的 API Key | sk-xxxx… |
| 自定义地址 | 可选 | API 代理地址，留空使用供应商默认地址 | https://api.deepseek.com |
| 日限额 | 可选 | 每日最大调用次数，0 = 无限 | 1000 |
| 备注 | 可选 | 辅助说明 | |

填写完成后点击 **「确定」**，配置即刻生效。

## 配置清单（建议）

建议为以下功能分别创建配置：

| 功能 | 建议模型 | 参考日限额 |
|------|----------|-----------|
| AI跑步教练 | deepseek-chat | 1000 次/日 |
| AI跑后总结 | moonshot-v1 | 500 次/日 |
| AI路线推荐 | qwen-turbo | 800 次/日 |
| AI帮写评论 | deepseek-chat | 2000 次/日 |
| AI找搭档 | deepseek-chat | 500 次/日 |
| AI每日金句 | qwen-turbo | 100 次/日 |
| AI路线审核 | deepseek-chat | 无限 |

> 同一个密钥可以被多个功能共用（如上表中的 deepseek-chat 密钥，跑步教练、帮写评论、找搭档、路线审核可共用）。

## 日常操作

### 启用/禁用功能
点击功能对应行的 **开关**：
- 绿色 = 已启用 → App 端可调用
- 灰色 = 已禁用 → App 端不可调用

**说明：** 禁用后功能从前端隐藏，不再消耗 API 配额。

### 轮换密钥
当密钥泄露或需要更换时：
1. 点击该行 **「轮换」** 按钮
2. 在弹出的对话框中输入 **新密钥**
3. 点击确定，旧密钥立即失效

### 编辑配置
点击 **「编辑」**，可修改模型、日限额、备注等字段。
> 编辑时密钥字段为空（需重新输入），若不修改密钥则留空保存。

### 删除配置
点击 **「删除」**，确认后该功能的密钥配置被移除。

## 日限额说明

- **进度条展示：** 显示当日已用量占总量的百分比
- **>80% 红色警告：** 接近限额，建议关注
- **无限配額：** 日限额设为 `0` 时，显示「无限」
- **计数规则：** 每日 00:00 自动重置

## 排障指南

| 现象 | 原因 | 处理 |
|------|------|------|
| 侧栏不显示菜单 | 浏览器缓存 | Ctrl+Shift+R 强制刷新 |
| 点击菜单空白 | 容器未重启 | 联系运维重启 stridemoor-admin 容器 |
| 提示「未授权」 | 缺少权限 | 检查 sys_menu 是否已分配角色 |
| 密钥保存失败 | 必填项未填 | 检查功能名称/供应商/密钥是否为空 |
| 日限额进度不变 | 今日无调用 | 正常现象，有调用后自动更新 |
| 开关无法切换 | 后端异常 | 查看容器日志定位错误 |
| 轮换后旧密钥仍可用 | CDN 缓存 | 等待 1-2 分钟或清 CDN 缓存 |

## 数据安全

- **密钥加密存储：** 所有 API Key 使用 AES 加密写入数据库
- **脱敏展示：** 页面只显示前4后4位（sk-aX…3Xv）
- **仅管理台可操作：** 密钥管理入口受 RuoYi 权限控制（`stridemoor:ai:key:*`）
- **App 端不存储明文密钥：** 运行时临时读取，不持久化
