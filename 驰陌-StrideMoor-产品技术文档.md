# 驰陌 / StrideMoor — 社交跑步教练 APP 产品技术文档

> 核心理念：**"驰于阡陌，自在奔跑"**
> 以路线为锚点，用数据对比帮助朋友改进跑步技术，有目标地跑步。

---

## 一、产品定位

**英文 Slogan：** *Stride in Moor, Run at Ease*

| 项目 | 描述 |
|------|------|
| 产品名 | 驰陌 / StrideMoor（暂定） |
| 一句话定位 | 社交跑步教练——分享路线，导入收藏，陪跑进步 |
| 目标用户 | 跑步爱好者（初级~中级），有运动社交需求 |
| 核心差异 | 不做赛事/跑团，以**路线**为锚点建圈子——围绕路线分享、收藏、陪跑形成社交闭环 |

## 二、核心功能模块

### 2.1 运动记录（基础层）

实时采集跑步过程中的全部数据，是后续对比和诊断的基础。

#### 🗺 地图与轨迹

| 功能 | 说明 | 技术要点 |
|------|------|---------|
| 实时轨迹绘制 | 跑步中在地图上实时画线 | GPS 定位 + 地图 SDK |
| 轨迹回放 | 跑完后 3D/2D 回放轨迹 | 轨迹数据存储 + 动画渲染 |
| 路线保存 | 将轨迹保存为"路线"（可被他人挑战） | 轨迹简化算法（Douglas-Peucker） |
| **路线分享** | 一键分享路线卡片到微信/QQ/链接，含路线地图+关键数据+难度标签 | 深度链接(Deeplink) + 分享SDK |
| **路线导入** | 从他人分享/社区发现/外部GPX文件导入路线到自己的收藏 | GPX解析 + 轨迹标准化 |
| **路线收藏** | 收藏感兴趣的路线，建立个人跑迹，按距离/难度/位置分类管理 | 收藏夹CRUD + 标签系统 + LBS推荐 |
| 路线导航 | 沿路线跑，语音导航，偏航提醒 | 实时偏航检测 + TTS |

#### 📊 实时数据采集

| 数据 | 采集方式 | 精度要求 |
|------|---------|---------|
| **配速** | GPS 位移 / 时间 | 每秒采样，5秒滑动平均 |
| **步频** | 手机加速度计（计步器）或连接运动手表 | 每步检测，每分钟统计 |
| **步幅** | GPS距离 / 步数（间接计算） | 依赖GPS+步频，需平滑滤波 |
| **心率** | 蓝牙连接运动手表/手环/心率带 | 每秒或每5秒更新 |
| **海拔/爬升** | 手机气压计 + GPS高程 | 气压计为主，GPS辅助校准 |
| **卡路里** | 基于体重+配速+心率估算 | 仅供参考 |

#### ⌚ 多设备支持

| 设备类型 | 连接方式 | 可获取数据 |
|---------|---------|-----------|
| Apple Watch | HealthKit 同步 / WatchOS 伴生APP | 心率、步频、步幅、GPS轨迹 |
| Garmin | Garmin Connect API（OAuth） | 全量数据 |
| 华为手环/手表 | 华为运动健康 Kit | 心率、步频、GPS轨迹 |
| 小米手环 | Zepp Life / 小米运动 API | 心率、步频 |
| 心率带（Polar/H10等） | BLE 蓝牙直连 | 高精度心率 |
| 其他 BLE 设备 | 通用 BLE HRM 协议 | 心率 |

### 2.2 路线生态（社交层）

核心理念：**分享 → 导入收藏 → 陪跑**，完整闭环

```
跑者A跑完路线 → 保存为路线 → 分享路线卡（含地图/数据/难度）
    ↓
跑者B看到 → 导入收藏到自己跑迹 → 随时可开始陪跑
    ↓
跑者B到起点 → 选择陪跑模式 → 实时影子陪跑 + 语音解说
    ↓
跑者B完成 → 自动生成对比报告 → 诊断建议 → 分享进步
    ↓
跑者B也分享路线 → 更多朋友加入 → 路线热度增长
```

#### 路线分享

| 功能 | 说明 |
|------|------|
| 分享路线卡 | 一键生成精美路线卡片（地图缩略图+距离+爬升+难度标签+创造者信息），分享到微信/QQ/朋友圈 |
| 深度链接 | 点击分享链接直接打开APP进入路线详情页（未安装跳应用商店） |
| 分享到社区 | 路线发布到驰陌 / StrideMoor社区，按城市/距离/难度浏览 |
| GPX导出 | 导出路线GPX文件，方便在Garmin等设备上使用 |
| GPX导入 | 从外部导入GPX文件（从其他APP/手表导出的路线），自动识别并标准化 |

#### 路线收藏与管理

| 功能 | 说明 |
|------|------|
| 收藏路线 | 一键收藏感兴趣的路线到个人跑迹 |
| 跑迹 | 个人收藏的所有路线，按列表/地图模式查看 |
| 智能分类 | 按距离（3km/5km/10km/半马/全马）、难度（轻松/中等/挑战）、位置（附近/收藏城市）自动标签 |
| 路线热度 | 每条路线显示被陪跑次数、评分、难度评级 |
| 距离排序 | 按离当前位置距离排序，优先推荐附近的收藏路线 |
| 待跑清单 | 标记"想去跑"，建立跑步愿望清单 |

#### 路线发现

| 功能 | 说明 |
|------|------|
| 附近热门 | 基于LBS推荐附近跑者常跑的路线 |
| 好友路线 | 好友最近跑过的路线，一键导入收藏 |
| 路线挑战榜 | 同一路线的陪跑成绩排行榜（不是竞速，是进步排行） |
| 难度推荐 | 根据你的历史数据推荐适合难度的路线 |

#### 路线排行榜

| 功能 | 说明 |
|------|------|
| 综合排名 | 同一条路线按总用时排名，显示 Top10 + 自己的排名 |
| **数据规则** | 每人只保留该路线的最新一次跑步记录用于排名，确保公平 |
| 打卡次数 | 记录跑过该路线的人数和总次数 |
| 一键挑战 | 从排行榜选择任意跑者，点击"向他挑战"，选择挑战目标维度后生成陪跑邀请 |
| **挑战目标** | 不止竞速，可选择：总用时/平均心率/平均步频/平均步幅等维度，达到对手水平即算成功 |
| 进步排名 | 可切换为"进步幅度"排行（对比自己历史最佳），鼓励进步而非竞速 |

#### 👻 陪跑模式（核心体验）

用户从收藏跑迹选择一条路线，点击"开始陪跑"，即可体验与原跑者的影子对决。

**影子陪跑（跑伴 Run）**

地图上显示原跑者的"跑伴"标记，实时同步移动。当用户跑过某个位置时，跑伴也显示在该位置原跑者当时的状态，形成直观的视觉对比。

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **真实回放** | 跑伴严格按原跑者的实际配速推进，全程体验原跑者的节奏变化 | 体验朋友的真实跑法，学习节奏控制 |
| **目标挑战** | 选择挑战维度（配速/心率/步频/步幅），跑伴按该维度的对手水平推进，达标即成功 | 不比速度比技术，不同水平也能公平挑战 |
| **匀速目标** | 跑伴以原跑者的平均配速匀速前进 | 作为稳定目标，训练节奏一致性 |
| **兔子模式** | 跑伴比原跑者快5%，始终在前方引导 | 冲击更好成绩，有目标感 |
| **龟兔模式** | 跑伴前半程比原跑者快，后半程比原跑者慢（负分段） | 练习负分段策略，前慢后快 |

**多人影子**

支持同时显示多个跑伴（多位朋友的同路线数据），每个人用不同颜色区分。可以和跑得最快的朋友比，也可以和水平接近的朋友比。

#### 🔊 语音播报系统（核心体验）

**三种播报模式**

| 模式 | 场景 | 播报内容 |
|------|------|----------|
| **独自跑** | 跑自己的路线，无对手 | 自身数据播报（配速/心率/步频/距离/用时） |
| **陪跑** | 跑他人路线，影子跟随 | 自身数据 + 与跑伴的对比数据 |
| **挑战跑** | 向排行榜对手发起挑战 | 自身数据 + 实时达标/未达标提醒 |

**播报频率（可自定义）**

| 选项 | 说明 |
|------|------|
| 每200m | 高频播报，适合短距离/间歇训练 |
| 每500m | 中频，适合5-10km |
| 每1000m | 低频，适合长距离/半马/全马 |
| 每5分钟 | 按时间播报，适合不看距离的跑者 |
| 每10分钟 | 低频按时间，适合长距离 |
| 仅异常 | 只在配速/心率偏离时播报，最安静 |

> 💡 建议勾选3-5项播报内容，超出时系统按优先级轮播，避免单次播报过长影响体验。

### 2.3 技术诊断（教练层）

#### 多维对比看板

| 维度 | 对比方式 | 诊断意义 |
|------|---------|---------|
| 分段配速 | 1km 切分，逐段对比 | 找出掉速段落 |
| 配速曲线 | 全程配速变化折线 | 前快后慢 = 起步策略问题 |
| 步频曲线 | 全程步频变化 | 步频波动大 = 节奏不稳 |
| 步幅曲线 | 全程步幅变化 | 步幅过大 = 容易受伤 |
| 心率曲线 | 全程心率变化 | 同配速心率更高 = 体能效率低 |
| 心率/配速比 | 每段心率÷配速 | 效率指标，越低越好 |
| 上坡表现 | 爬升段配速衰减对比 | 腿部力量/爬坡技术 |
| 下坡表现 | 下降段配速对比 | 下坡技术/胆量 |

#### 智能诊断建议

根据对比数据自动生成改进建议：

```
📋 跑步诊断报告

✅ 优势：
- 前5km配速稳定，节奏控制好
- 步频保持170+，跑步效率高

⚠️ 待改进：
- 第6-8km步频从172降到158，可能疲劳导致步幅放大
  → 建议：后半程主动缩小步幅，维持高步频
- 上坡段配速掉了45秒/km，衰减比A大20秒
  → 建议：加强坡道训练，上坡减小步幅提高步频
- 心率在后半程偏高，同配速下比A高12bpm
  → 建议：增加有氧基础训练，提升乳酸阈值

🎯 下次目标：
- 保持前5km节奏
- 6km后步频不低于165
- 上坡配速衰减控制在30秒以内
```

### 3.2 关键技术选型

#### 地图与轨迹

| 组件 | 方案 | 理由 |
|------|------|------|
| 地图 SDK（国内） | 高德地图 SDK | 国内定位精度高，合规，免费额度够用 |
| 地图 SDK（海外） | MapBox | 全球覆盖，自定义样式能力强 |
| 轨迹渲染 | 自定义 Overlay + OpenGL | 高性能绘制大量轨迹点 |
| 离线地图 | MVP 可不做 | 先保证在线体验 |
| 轨迹匹配 | 自研或用 Valhalla map-matching | 将GPS点匹配到道路，消除漂移 |

#### 实时数据采集

| 数据 | 移动端方案 | 精度优化 |
|------|-----------|---------|
| GPS位置 | 系统定位服务（CLLocationManager / FusedLocationProvider） | 请求高精度模式，过滤静态漂移点 |
| 步频 | 加速度计 + 步行检测API | iOS: CMPedometer / Android: Step Counter Sensor |
| 步幅 | GPS距离 ÷ 步数 | Kalman滤波平滑，避免GPS抖动导致异常值 |
| 心率 | BLE 4.0 连接心率设备 | 标准 HRM Profile (0x180D) |
| 海拔 | 气压计 + DEM数字高程 | 气压计为主，DEM校准 |

#### 路线匹配算法

核心问题：两个人跑同一条路，GPS轨迹不会完全重合，需要判断是否"同一路线"。

```
算法流程：
1. 轨迹简化 → Douglas-Peucker 算法压缩到关键转折点
2. 起终点匹配 → 起终点距离 < 100m
3. 路线形状匹配 → DTW（动态时间规整）计算轨迹相似度
4. 距离匹配 → 总距离偏差 < 10%
5. 相似度 > 85% → 判定为同一路线

分段对齐：
- 将路线按1km切分
- 用 Frechet 距离匹配对应分段
- 确保"第3km"对齐的是同一段路
```

#### 数据存储

| 数据类型 | 存储方案 | 理由 |
|---------|---------|------|
| 用户/社交数据 | PostgreSQL | 关系型，事务，成熟 |
| 跑步时序数据 | TimescaleDB | PostgreSQL扩展，高效存储每秒采样数据 |
| 轨迹原始数据 | 对象存储(S3/OSS) + TimescaleDB索引 | 原始GPX文件存S3，关键指标存时序库 |
| 排行榜/缓存 | Redis (Sorted Set) | 实时排行，高性能 |
| 全文搜索 | Elasticsearch | 搜索路线名称/标签 |

### 3.3 核心数据模型

```sql
-- 用户
users: id, nickname, avatar, device_info, created_at

-- 路线（由跑步轨迹抽象而来）
routes: id, creator_id, name, geometry(simplified), 
        distance, elevation_gain, difficulty, popularity,
        start_point, center_point

-- 跑步记录
runs: id, user_id, route_id(nullable), 
      start_time, end_time, total_distance, total_time,
      avg_pace, avg_heart_rate, avg_cadence, avg_stride_length,
      elevation_gain, calories, weather, temperature,
      gpx_file_url, device_type

-- 跑步分段数据（每1km）
run_splits: id, run_id, split_index, 
            distance, time, pace, 
            avg_heart_rate, avg_cadence, avg_stride_length,
            elevation_gain, elevation_loss

-- 跑步秒级时序数据（存TimescaleDB）
run_samples: run_id, timestamp, 
             latitude, longitude, altitude,
             pace, heart_rate, cadence, stride_length,
             distance_from_start

-- 路线收藏
route_favorites: id, user_id, route_id, 
                 tag(想去跑/已跑过), created_at

-- 路线排行榜快照（每次跑步后更新）
route_leaderboards: id, route_id, user_id,
                    run_id,  -- 关联最新跑步记录
                    total_time, avg_pace, recorded_at

-- 路线挑战（陪跑）
challenges: id, route_id, 
            challenger_id, challenger_run_id,
            invitee_id, ghost_mode(real_replay/constant/rabbit/tortoise_hare/goal),
            goal_metric(pace/heart_rate/cadence/stride_length),  -- 目标挑战维度
            status(pending/running/completed),
            created_at

-- 对比报告
comparisons: id, challenge_id,
             run_a_id, run_b_id,
             overall_diff, 
             splits_json,  -- 分段对比详情
             diagnosis_json  -- AI诊断建议
```

## 五、技术风险与难点

| 难点 | 风险等级 | 应对策略 |
|------|---------|---------|
| GPS精度/漂移 | 🔴高 | Kalman滤波 + 地图匹配 + 静态点过滤 |
| 室内/隧道GPS丢失 | 🟡中 | 加速度计推算补位，标注"估算段" |
| 步幅计算不准 | 🟡中 | 多源融合（GPS+加速度计），平滑滤波 |
| 路线匹配算法 | 🔴高 | 先用起终点+距离粗筛，再用DTW精匹配 |
| BLE心率连接稳定性 | 🟡中 | 重连机制 + 缓存最近心率 + 断连提示 |
| 电池续航 | 🔴高 | GPS高精度模式耗电大，需优化采样策略 |
| 影子同步精度 | 🔴高 | 路线距离插值+GPS位置匹配，而非时间线对齐 |
| 实时语音播报延迟 | 🟡中 | 预加载语音包，TTS本地化 |

| 特性 | 华为运动健康 | 第一赛道 | Keep | 悦跑圈 | NRC | Strava | RQrun | **驰陌 / StrideMoor** |
|------|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| GPS轨迹 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 步频/步幅 | ✅ | ⚠️ | ❌ | ⚠️ | ✅ | ✅ | ⚠️ | ✅ |
| 路线分享 | ❌ | ✅ | ❌ | ⚠️ | ❌ | ✅ | ❌ | **✅核心** |

| **实时语音对比** | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | ❌ | **✅核心** |
| **技术诊断** | ⚠️AI教练 | ❌ | ❌ | ❌ | ⚠️指导 | ❌ | ❌ | **✅核心** |

| 赛事服务 | ❌ | ✅ | ❌ | ✅ | ⚠️ | ❌ | ❌ | ❌不做 |
| 设备生态 | 华为 | 通用 | 多设备 | 通用 | Apple | 通用 | Apple | 多设备 |

**核心差异**：现有产品要么重赛事（第一赛道），要么重社区（Keep/悦跑圈），要么重设备（华为）。没有人围绕"路线"做社交闭环。驰陌 / StrideMoor不做赛事、不做跑团，专注**路线**——围绕路线分享、收藏、陪跑，让路线本身成为社交纽带。

**重点参考**：
- **第一赛道**：路线分享+排行榜的思路值得借鉴，但我们不走赛事服务路线
- **华为运动健康**：设备生态、数据可视化、简洁体验值得参考

### 8.3 模块一：发现（首页）

**定位**：内容入口+个人记录，让用户看到动态、找到跑迹、被激发去跑。

| 页面 | 功能点 |
|------|--------|
| **首页 Feed** | 跑友动态流 / 推荐挑战 / 官方活动 |
| **运动记录** | 个人所有跑步记录列表，按时间倒序；显示距离/配速/用时；点击进入跑步详情 |
| **跑步详情** | 单次跑步的完整数据：轨迹地图 + 分段配速 + 心率曲线 + 步频/步幅 + 爬升；如有陪跑则展示对比 |
| **跑迹广场** | 浏览所有跑迹，查看我参与过的跑迹；每条跑迹显示打卡排行榜，可选择跑伴发起挑战；类似华为运动健康的"路线"体验 |
| **跑迹详情** | 跑迹地图 + 数据（距离/爬升/难度）+ 打卡排行榜 + 参与人数 + 一键挑战 |
| **跑友动态** | 所有跑友可发动态（无需关注/加好友）；跑步成绩、跑迹分享、挑战结果、个人感悟；点赞/评论 |
| **跑迹搜索** | 按城市/距离/难度/关键词搜索跑迹 |
| **附近跑迹地图** | 地图模式查看周边所有跑迹，点击查看详情 |

### 8.4 模块二：运动（核心体验）

**定位**：运动全流程——从准备到结束，三种模式统一入口。

| 页面 | 功能点 |
|------|--------|
| **跑步准备页** | 选择模式（独自跑/陪跑/挑战跑）→ 选择路线 → 选择播报频率 → 勾选播报内容 → 连接设备（手表/心率带） |
| **GPS搜星页** | 实时显示GPS精度，信号稳定后显示"可以开始" |
| **跑步中主界面** | 实时地图（含跑伴标记）+ 核心数据面板（配速/距离/用时/心率）+ 底部播报快捷设置 |
| **跑步中锁屏** | 常亮显示核心数据（配速/距离），防止误触，上滑解锁 |
| **分段播报浮窗** | 播报时顶部弹出卡片，显示当前段数据+对比数据，3秒后自动收起 |
| **暂停/继续** | 暂停后显示本次已跑数据摘要 |
| **跑步结束页** | 完成动效 + 本次数据总览（用时/配速/心率/步频/步幅/卡路里/爬升）→ 保存为路线 → 分享 |

**独自跑流程**：
```
准备页 → 选路线(可选) → 搜星 → 开始 → 跑步中(语音播报个人数据) → 结束
```

**陪跑流程**：
```
准备页 → 从跑迹选路线 → 选陪跑模式(真实回放/匀速/兔子/龟兔) → 选播报内容 → 搜星 → 开始
→ 跑步中(地图跑伴+语音对比) → 结束 → 对比报告
```

**挑战跑流程**：
```
准备页 → 从排行榜选对手 → 选挑战维度(配速/心率/步频/步幅) → 选播报内容 → 搜星 → 开始
→ 跑步中(跑伴+达标提醒) → 结束 → 挑战结果(达标/未达标) + 对比报告
```
